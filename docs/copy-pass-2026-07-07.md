## Final replacement copy

### 1. Home

**Current issue**
The page is titled and described like a roadmap page, but it functions more like a move overview and current-state dashboard.

**Replace with**
- Eyebrow: `Move Map`
- Title: `Home`
- Description: `A calm overview of where the move stands, what needs attention next, and which family decisions are still open.`

**Usage note**
If the UI keeps the current title structure for now, the description should still change to this language.

### 2. Roadmap

**Current issue**
The current framing is broadly correct, but it should speak more directly to the page's role as the timeline view.

**Replace with**
- Eyebrow: `Roadmap`
- Title: `From research to arrival`
- Description: `A time-based view of the move, showing phases, milestones, dependencies, and what has to happen before January 2027.`

### 3. Conversation

**Current issue**
The copy under the header should reflect a conversation tool, not a trip concept.

**Replace with**
- Eyebrow: `Conversation`
- Title: `Family conversation tracker`
- Description: `A shared space to guide move conversations, track who has spoken, capture open questions, and save what still needs a follow-up.`

**Usage note**
Any copy currently using `The Big Trip` as the framing label under this page should be replaced with `Conversation` language.

### 4. Calendar

**Current issue**
The page should read as a practical family calendar for move-related timing, not a thematic timeline.

**Replace with**
- Eyebrow: `Calendar`
- Title: `Family move calendar`
- Description: `A month-by-month calendar for deadlines, family dates, move milestones, and the events that need everyone to stay in sync.`

### 5. Options

**Current issue**
The current title and description overemphasize local editing behavior instead of the page purpose.

**Replace with**
- Eyebrow: `Options`
- Title: `Compare possible paths`
- Description: `Use this page to compare visa routes, neighborhoods, schools, housing, and other planning options before deciding.`

**Usage note**
Avoid references to browser state, local editors, or seeded data in user-facing copy.

### 6. Ideas

**Current issue**
The current framing sounds vague and indirect. It should clearly describe a place for prompts, questions, and emerging thoughts.

**Replace with**
- Eyebrow: `Ideas`
- Title: `Keep track of open ideas`
- Description: `Capture questions, conversation prompts, and early thoughts worth revisiting before they turn into decisions or tasks.`

**Usage note**
Avoid the phrase `Questions worth keeping warm.` It feels more poetic than practical and was specifically called out by the clarified feedback.
